# Kitchen Ticket Ergonomics Review

Review the ticket like someone with grease on their hands and three orders waiting.

## Checklist

- item lines stand out first
- channel and delivery mode do not bury item data
- observations are visible but not noisy
- priority markers are obvious
- repeated info is removed

## Tool Doctrine

- Use `rtk` for shell.
- Use `m1nd` if ticket content is generated in multiple backend locations.
- Use `graphify` to find every ticket-related file when the graph exists.
- Use `Ponytail full` to shorten the ticket until only useful lines remain.
